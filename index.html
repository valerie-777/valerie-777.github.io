> Attacktive Directory

> It's a foundational lab focusing on attacking a vulnerable Windows Active Directory (AD) environment. It simulates a corporate network to teach domain enumeration and credential extraction.
> I began by performing initial enumeration to map the network and found valid domain users. Next, I abused Kerberos pre-authentication via AS-REP Roasting to extract ticket hashes. After cracking those hashes offline, I used the compromised credentials to dump the Domain Controller's NTDS.dit file, granting me full administrative control over the domain.
> Tools Used: Nmap, Enum4linux, Kerbrute, Impacket (GetNPUsers.py, secretsdump.py), BloodHound, and Hashcat.
> Identifying active AD services, understanding the danger of user accounts that do not require Kerberos pre-authentication, and seeing firsthand how Pass-the-Hash and credential dumping lead to total domain compromise.

> Sweettooth Inc.

> It's a web and container exploitation lab. It requires gaining an initial foothold on a server running an InfluxDB database and subsequently breaking out of a Docker container.
> I started by enumerating open ports to identify the InfluxDB instance. By forging a JWT (JSON Web Token), I gained unauthorized access to extract SSH credentials. Once inside the Docker container, I abused an exposed Docker Engine API (often found on port 8080) to mount the host filesystem and execute a reverse shell, fully compromising the underlying host machine.
> Tools Used: Nmap, curl, Influx CLI, Docker, Netcat.
> Key Lessons Learned: Exploiting JWT vulnerabilities, identifying containerized environments, and understanding how Docker Engine API misconfigurations can lead to container escapes and host privilege escalation.
