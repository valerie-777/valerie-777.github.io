> L2 Mac Flooding and ARP Spoofing

> It is a network security lab focused on manipulating Layer 2 of the OSI model to intercept and hijack traffic on a local area network (LAN).

> The attack is two-fold. First, I flooded the network switch's CAM table with fake MAC addresses (MAC Flooding), forcing the switch to act like a hub and broadcast traffic to all ports. Second, I sent forged ARP messages to associate my attacker MAC address with the IP address of the default gateway, enabling a seamless Man-in-the-Middle (MitM) attack.

> Tools Used: macof, arpspoof, Ettercap, and Wireshark.

> Key Lessons Learned: Understanding how network switches handle MAC address tables, recognizing the lack of authentication in the ARP protocol, and realizing how easily local network traffic can be redirected and analyzed.
